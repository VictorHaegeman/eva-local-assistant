# Gmail Triage

Use this skill for Gmail, inbox, replies, appointments, apartment emails, DreamLense emails, unanswered threads, or calendar-related emails.

Operating pattern:

1. Read real Gmail API data first.
2. Separate human/actionable emails from newsletters, ads, notifications, receipts, alerts, and noreply messages.
3. For a requested email, open or reference the exact thread/message before drafting.
4. Check whether Victor already replied in the thread.
5. Draft in the same language as the incoming email unless Victor asks otherwise.
6. Use Victor's previous sent replies only as style/examples, not as facts to invent.
7. If the email implies an appointment, propose a Calendar action but do not create it without an allowed flow.

Auto-reply limits:

- Only short, obvious, non-sensitive replies can be sent automatically.
- Never auto-send for legal, financial, housing commitment, medical, conflict, payment, password, account access, or ambiguous intent.
- If unsure, create a Gmail draft and ask for validation.

Never invent an email subject, sender, listing detail, or calendar event.
